Docflix Media Suite

User Manual — Version 2.0.5

Docflix Media Suite is a batch video converter that encodes files to H.265/HEVC (and other codecs) using ffmpeg, with support for CPU and multi-GPU encoding (NVIDIA NVENC, Intel QSV, AMD VAAPI). It includes a full-featured desktop GUI, a subtitle editor with OCR and Whisper-based auto-sync, a TV show renamer, a media processor, and a headless CLI tool.

2. Quick Start

  1. Launch the app via the system menu, terminal (docflix), or by running python3 video_converter.py
  2. Add files — drag and drop video files onto the window, or use Ctrl+O to open files / Ctrl+Shift+O to open a folder
  3. Choose settings — select your encoder (CPU or GPU), video codec, quality mode, and container format in the settings panel
  4. Click "Start Conversion" — the app will convert all queued files with your chosen settings
  5. Monitor progress — a progress bar and log show real-time encoding status with batch ETA
For a quick test before committing to a full conversion, select a file and press Ctrl+T to encode just the first 30 seconds with your current settings.

Settings Panel

The collapsible settings panel (Ctrl+Shift+S to toggle) contains all encoding controls:

  • Video Codec — H.265/HEVC, H.264/AVC, AV1, VP9, MPEG-4, ProRes, or Copy (no re-encode)
  • Container — .mkv, .mp4, .ts, or match source
  • Transcode Mode — Video Only, Audio Only, or Both
  • Quality Mode — Bitrate (fixed size) or CRF (constant quality)
  • Bitrate slider — 1M to 16M with quick-set buttons
  • CRF value — lower = better quality, higher = smaller file
  • Preset — speed/quality trade-off (ultrafast to veryslow for CPU; p1-p7 for NVENC)
  • Audio — codec and bitrate dropdowns
  • Options — Skip existing, Delete originals, HW Decode, Two-pass, Verify output
  • Metadata — Strip chapters, Strip tags, Set track metadata with language codes
  • Edition — Tag videos with version info (Theatrical, Director's Cut, etc.)
  • Chapters — Auto-generate chapter markers at a specified interval

File Queue

The file queue shows all files to be converted. You can:

  • Add files — drag and drop, Ctrl+O (files), Ctrl+Shift+O (folder)
  • Remove files — select and press Delete, or right-click → Remove
  • Double-click — opens the Internal Subtitles dialog
  • Right-click — context menu with play, override settings, subtitle management, Enhanced Media Details, and remove options

Starting a Conversion

  1. Add files to the queue
  2. Adjust settings as needed (or use per-file overrides for individual files)
  3. Click Start Conversion
  4. Use Pause / Stop to control the process
  5. When complete, a sound notification plays (configurable in Default Settings)
  6. Click Clear Finished to remove completed files, leaving failed files for retry
The Batch ETA in the status bar shows estimated time remaining for all files, based on rolling average encoding speed.

Quality Modes

Bitrate Mode (Fixed Size)

Sets a target bitrate. The output file size is predictable. Use the slider (1M–16M) or type a value. Quick buttons: 1M, 2M, 3M, 4M, 8M, 16M.

CRF Mode (Constant Quality)

CRF (Constant Rate Factor) maintains consistent visual quality throughout the file. The file size varies based on content complexity.

CRF ValueQualityFile Size
18Visually losslessLarge
23Good quality (default)Medium
28Acceptable qualitySmall
35+Noticeable quality lossVery small
For GPU encoders, the quality flag varies: NVENC uses -cq, QSV uses -global_quality, VAAPI uses -qp. The app handles this automatically.

Presets

CPU Presets (libx265 / libx264)

ultrafast → superfast → veryfast → faster → fast → medium → slow → slower → veryslow

Slower presets produce smaller files at the same quality, but take much longer to encode.

NVIDIA NVENC Presets

p1 (fastest) → p4 (default) → p7 (best quality)

Intel QSV Presets

veryfast → medium (default) → veryslow

Two-Pass Encoding

Two-pass encoding analyzes the video in a first pass and optimizes bit allocation in the second pass. This produces better quality at a given bitrate, especially for complex scenes. Available for CPU encoding; GPU encoders use multipass where supported.

Two-pass roughly doubles encoding time. It's most beneficial in bitrate mode; CRF mode already adapts per-frame.

Hardware Decode

When a GPU encoder is selected, the HW Decode checkbox enables hardware-accelerated decoding. This speeds up encoding by offloading both decoding and encoding to the GPU.

HW Decode is automatically disabled when burn-in subtitles are used, as subtitle rendering requires CPU-side video filtering. It may also cause issues with some source files (variable resolution, unusual codecs). Uncheck it if you encounter errors.

6. Metadata & Tagging

Strip Chapters / Strip Tags

  • Strip chapters — removes all chapter markers from the output (-map_chapters -1)
  • Strip tags — removes all global metadata/tags from the output (-map_metadata -1)

Set Track Metadata

When enabled, sets language codes on video, audio, and subtitle tracks and clears track names and the container title. Default language codes:

TrackDefaultPurpose
Video (V:)undUndetermined
Audio (A:)engEnglish
Subtitle (S:)engEnglish

Edition Tagging

Tag videos with version/edition information. The edition is written to the container title metadata field, which is displayed by most media players and servers (VLC, Plex, Kodi, Jellyfin).

Presets

Theatrical, Director's Cut, Extended, Extended Director's Cut, Unrated, Special Edition, IMAX, Criterion, Remastered, Anniversary Edition, Ultimate Edition, or Custom... (free text entry).

Plex Filename Tag

When "Add to filename (Plex)" is checked, the output filename includes a {edition-Director's Cut} tag that Plex recognizes for grouping different editions of the same movie.

Example: Superman {edition-Director's Cut}-2M-H265_ultrafast.mkv

The edition tag works independently of "Set track metadata". If both are active, the edition overrides the title-clearing behavior.

Add Chapters

Auto-generate evenly-spaced chapter markers at a configurable interval (1–60 minutes, default 5). Chapters are injected via a temporary FFMETADATA1 file.

"Add chapters" and "Strip chapters" are mutually exclusive — checking one automatically unchecks the other.

Chapters make it easy to jump through a long video. Most players (VLC, mpv, Plex, Kodi) display chapter markers in the seek bar.

7. Subtitles

Internal Subtitles

Double-click a file in the queue (or right-click → Internal Subtitles) to manage subtitle tracks already embedded in the source file. For each track you can:

  • Keep or drop the track
  • Convert to a different format (SRT, ASS, WebVTT, TTML)
  • Extract to a standalone file
  • Edit in the built-in subtitle editor

Bitmap subtitles (PGS, VobSub) can be converted to text via OCR when set to a text format.

External Subtitles

Attach external subtitle files to videos:

  • Drag and drop .srt, .ass, .ssa, .vtt, .sub, .idx, .sup files onto the queue
  • Auto-matching — subtitles are matched to videos by filename stem
  • Auto-detection — language, forced flag, SDH flag detected from filename (e.g., movie.eng.forced.srt)

Embed vs Burn-In

ModeDescriptionCan be toggled?
EmbedMuxed as a selectable subtitle streamYes (player controls)
Burn-inRendered permanently onto the videoNo (permanent)

Subtitle Editor

A full-featured subtitle editor accessible from Tools → Subtitle Editor (standalone) or by editing an internal/external subtitle track. Features:

  • Inline editing — double-click any cell to edit text directly
  • Filters — Remove HI, Remove Tags, Remove Ads/Credits, Remove Stray Notes, Remove Leading Dashes, Remove ALL CAPS HI, Remove Off-Screen Quotes, Remove Duplicates, Merge Short Cues, Reduce to 2 Lines, Fix ALL CAPS
  • Search & Replace — with Find, Replace, Replace All, and wrap-around
  • Search/Replace List — persistent find/replace pairs for common corrections
  • Timing tools — offset (shift ±ms) and stretch (scale by factor)
  • Split / Join / Insert cues via right-click context menu
  • Undo/Redo (Ctrl+Z / Ctrl+Y)
  • Video preview at cue timestamp via ffplay
  • Color-coded rows — yellow=modified, blue=HI, pink=tags, orange=long lines, green=search match
  • Save to Video — re-mux edited subtitle directly back into the video (no re-encoding)

Video Subtitle Editing

Drag a video file onto the standalone editor to extract an internal subtitle stream, edit it, and re-mux it back into the video. The stream picker dialog lets you choose which track to edit.

Smart Sync (Whisper-based Auto-Sync)

Automatically sync subtitles to video audio using speech recognition. Access via Timing → Smart Sync in the subtitle editor.

Engines

EngineAccuracySpeedSize
Standard (faster-whisper)~400msFast (CPU)~200MB
Precise (WhisperX)~50msFaster with GPU~2GB

Scan Modes

  • Quick Scan — samples segments across the video for fast offset detection
  • Full Scan — transcribes entire audio for maximum anchor points
  • Direct Align (WhisperX only) — skips transcription, aligns subtitle text directly against audio

Apply Methods

  • Apply Sync — applies a single global offset (median of matched pairs)
  • Re-time All — per-cue adjustment using piecewise linear interpolation from anchor points

Bitmap Subtitle OCR

Convert PGS (Blu-ray) and VobSub (DVD) bitmap subtitles to text SRT format using Tesseract OCR. Activated when a bitmap track is set to a text format in the Internal Subtitles dialog.

  • Single-pass rendering — overlays subtitle stream on a black canvas (~2 min for 1-hour episode)
  • Parallel OCR — uses multiple CPU cores simultaneously
  • Smart cropping — crops to subtitle text region before OCR (~13x fewer pixels)
  • Live monitor window — real-time progress, image preview, scrolling cue list
  • Post-processing — comprehensive fixes for common OCR mistakes (| → I, music note detection, etc.)

Spell Checker

Interactive spell check (F7 or Tools → Spell Check) that scans cues one at a time with Replace, Replace All, Skip, Ignore, Add to Dict, and Add as Name buttons. Custom dictionary persisted to preferences.

Batch Filter

Process multiple subtitle files at once (Tools → Batch Filter). Drag and drop files, select filters via checkboxes, and run. Includes batch Search & Replace with persistent find/replace pairs.

8. Tools

Media Processor

A standalone remux-only post-processing tool (Tools → Media Processor, Ctrl+M). Processes already-encoded files without re-encoding (-c:v copy).

Operations

  • Convert audio — change audio codec (AAC, AC3, EAC3, MP3, Opus, FLAC, Copy) + bitrate
  • Strip chapters / tags / existing subtitles
  • Mux external subtitles — auto-detects *.eng.srt and *.eng.forced.srt alongside videos
  • Set track metadata — language codes, clear titles
  • Edition tagging — same presets as the main converter
  • Add chapters — auto-generate at configurable intervals
  • Parallel processing — concurrent ffmpeg execution across multiple CPU cores
  • Output options — replace in-place or save to a separate folder
  • Per-file overrides — right-click any file for individual settings
All operations are combined into a single ffmpeg command per file, making it very efficient.

TV Show Renamer

Batch rename TV show and movie files using TVDB or TMDB metadata (Tools → TV Show Renamer).

  • Auto-detection — detects show/movie names from filenames and auto-searches the selected provider
  • Disambiguation — when multiple matches are found, shows a card-based picker with poster thumbnails and synopsis
  • Movie support — movies renamed to Name (Year).ext
  • Multi-episode — handles S01E01E02 and S01E01-E03 patterns
  • Date-based episodes — detects 2026.04.22 patterns for daily shows
  • Subtitle preservation — preserves language/forced/SDH tags from original filenames
  • Undo — revert the last rename batch (Ctrl+Z)
  • Configurable template — Settings → Filename Template with flat and folder presets

Folder Templates

Use / in the filename template to automatically create folder hierarchies during rename. For example:

TemplateResult
{show} S{season}E{episode} {title}Breaking Bad S01E01 Pilot.mkv
{show}/Season {season}/{show} S{season}E{episode} {title}Breaking Bad/Season 01/Breaking Bad S01E01 Pilot.mkv
{show}/S{season}/{show} S{season}E{episode} {title}Breaking Bad/S01/Breaking Bad S01E01 Pilot.mkv

Folders are created relative to the source file’s location. Undo will move files back and clean up any empty folders that were created.

9. Per-File Overrides

Right-click a file → Override Settings to set different encoding parameters for individual files. Overrides include:

  • Encoder, video codec, quality mode, bitrate/CRF, preset
  • Audio codec and bitrate
  • Skip existing, delete originals, HW decode
  • Strip chapters, strip tags, set track metadata with language codes
  • Edition tag and Plex filename option

Files with overrides show a ⚙️ icon in the queue. Double-click to edit the override.

10. CLI Usage

The convert_videos.sh script provides headless batch conversion from the command line. Run it from the directory containing your video files:

# CPU encoding, default settings
./convert_videos.sh

# NVIDIA GPU encoding
./convert_videos.sh --gpu

# Intel QSV encoding
./convert_videos.sh --qsv

# AMD VAAPI encoding
./convert_videos.sh --vaapi

# CRF quality mode
./convert_videos.sh --crf 22

# GPU, high quality preset, overwrite existing
./convert_videos.sh --gpu --gpu-preset p5 --overwrite

12. Preferences

Preferences are auto-saved to ~/.local/share/docflix/preferences.json when the Default Settings dialog is closed. No manual save is required.

What is saved

  • Encoder, video codec, container, quality mode, CRF, presets
  • Audio codec and bitrate
  • All checkboxes (skip existing, delete originals, HW decode, two-pass, verify output, sound notification)
  • Metadata options (strip chapters/tags, track metadata, language codes)
  • Edition tag and Plex filename option
  • Chapter insertion settings
  • Media player preference
  • Recent folders, custom ad patterns, character names, spell check dictionary
  • TV Show Renamer API keys, provider, and template
  • Media Processor settings (all options)

What is NOT saved

  • Video bitrate — intentionally resets to 2M on every launch to prevent silent mismatches
  • Transcode mode — always starts in Video Only mode

To reset all settings: Settings → Reset to Defaults, or delete the preferences.json file.

13. Troubleshooting

Common Issues

GPU encoding fails

If a GPU encoder fails during conversion, the app automatically retries with CPU encoding. If you consistently get GPU failures:

  • Try unchecking HW Decode — some sources are incompatible with hardware decoding
  • Ensure your GPU drivers are up to date
  • Check that ffmpeg was compiled with your GPU's encoder support

Small or empty output files

  • Check the log for error messages
  • Try a different encoder (switch from GPU to CPU)
  • Try a different container format
  • Ensure the source file is not corrupted

Subtitles not appearing in output

  • MPEG-TS output — text subtitles are not supported in .ts containers; only DVB subtitles work
  • MP4 output — only mov_text format is supported; other formats are auto-converted
  • Check that "Strip existing subtitle tracks" is not enabled

App appears tiny on high-DPI monitor

The app includes automatic DPI scaling detection. If it's still too small:

  • Set the GDK_SCALE=2 environment variable before launching
  • Or set Xft.dpi: 192 in your X resources (~/.Xresources)

Drag and drop not working

Drag and drop requires tkinterdnd2. Install it with: pip install tkinterdnd2

Zenity folder dialogs not appearing

Install zenity: sudo apt install zenity. The app falls back to Tkinter's built-in dialog if zenity is not available.
